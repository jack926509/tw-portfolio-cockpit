# 台股配置控制台

分配規劃 × 持續買進 × 未來推估。
設定每月投入與配置，推估持續買進多年後資產的可能區間與達標機率，並可把股利全掃入複利目標。
單一 HTML 檔、零安裝，丟上 GitHub Pages 即可使用，可自行增刪任意股票或 ETF。

> ⚠️ 純規劃試算工具，非投資建議。所有報酬率皆為假設，不代表未來表現。

---

## 線上使用

部署到 GitHub Pages 後，網址格式為：

```
https://<你的帳號>.github.io/<repo 名稱>/
```

## 功能特色

- **投資計劃精靈與計劃書（右上「投資計劃書」）**：新手不知道怎麼設定時，回答 4 個簡單問題（每月可投入、投資多久、風險屬性保守／穩健／積極、選填目標金額），工具會依**你目前持有（含新增）的標的**自動配重，產生可讀的「投資計劃書」——白話的成果推估（中位數、P10–P90 區間、達標率）、風險提醒（最壞回檔）、配置與內扣費用侵蝕、每月行動清單。可「套用此計劃」（依類別權重重配並設好投入參數）或「只套用投入參數」。
- **依類別自動配重**：每檔標的會自動判斷類別（核心市值ETF／衛星主動／個股，新增標的時依產業別與費率帶入）；風險屬性決定各類別權重、同類平均分配，適用任何標的，不綁死特定幾檔。某類別沒有標的時權重自動併回其他類別。
- **未來資產推估（主畫面）**：開啟即見「以目前的投入與分配持續買進 N 年後」資產的機率區間。預設用台股歷史報酬做「平穩拔靴」（含波動、厚尾與序列報酬風險），輸出 P10／P50／P90 機率帶、中位數預期與達標率，比單一直線試算更貼近不確定性。可在「進階設定」切換參數化（GBM 對數常態）、報酬分布、相關係數、CAPE 估值參考、實質購買力、歷史回測校驗與股利稅負。
- **目標金額回推**：輸入想在第 N 年達到的金額，回推目前設定下需要的每月（或單筆）投入。
- **分配規劃**：設定每檔的配置比例、內扣費用、現金殖利率與年化波動度，推估即時反映。
- **新增標的即抓**：按「新增標的」跳出輸入框，填代號按確定即**立刻**抓取名稱、股價、現金殖利率、年化波動%與內扣%並計入推估（資料來自 FinMind，免註冊、免金鑰；抓不到的欄位保留原值）。名稱／股價／內扣%／現金殖利%／波動%五欄為自動帶入、已鎖定（灰底）不可手動編輯，要更新可重抓代號或按「更新股價」；你只需設定「配置%」並勾選「配息」。
- **股利全掃複利目標**：可指定一檔不配息標的（預設 009816）為複利目標，其餘標的的現金股利每年自動掃入該檔再投資，並顯示組成隨時間「漂移」的堆疊圖。
- **一鍵更新股價與殖利率**：按「更新股價」自動從台灣證交所／櫃買 OpenAPI 抓取最近收盤價與殖利率並回填，名稱空白時一併補上；大盤來源失敗時改走 FinMind 逐檔備援（皆免註冊、免金鑰）。
- **情境比較 A／B**：把兩組設定存成快照疊圖比較累積資產與期末差異。
- **零股買法**：依設定預算即時算出每檔買幾股、花多少錢。
- **設定匯出／匯入**：可把目前設定下載成 JSON 備份或分享，再從檔案匯入還原。
- **自動儲存**：設定存於瀏覽器 `localStorage`，下次開啟自動還原（載入時會驗證並補齊欄位，舊資料也能安全還原）。

## 參數說明

| 欄位 | 意義 |
|---|---|
| 股價 | 目前市價（元），用來換算零股股數 |
| 配置% | 該標的佔投入金額的比例；合計不滿 100% 會自動正規化 |
| 內扣% | 基金年度總費用率；無免金鑰 API，由內建費率表自動帶入並鎖定（常見 ETF 用公開資料估值、個股為 0、未收錄 ETF 給 0.4% 保守估值） |
| 現金殖利% | 年現金配息率，用於計算掃入複利目標的股利 |
| 波動% | 年化波動度，供進階「參數化」模型逐標的模擬用（市值型 ETF 約 18、主動型約 22、個股約 30）；歷史拔靴模型不使用此欄 |
| 配息 | 選股利再投入方式（兩種都會再投資）：打勾＝股利每年掃入你選的複利目標集中滾；取消＝股利留在原檔自己複利（原地再投入）。皆非把錢領出來花掉 |

## 試算模型與假設

為求透明，模型邏輯如下（刻意簡化）：

- 全部標的套用**同一個毛報酬率**假設，逐月複利。
- **複利目標**：以毛報酬扣自身內扣費用成長，並吸收其他標的掃入的股利。
- **配息標的**：只保留「價格報酬」（毛報酬 − 現金殖利率 − 內扣）成長，現金股利於每年底掃出至複利目標。
- **不配息且非目標的標的**：以毛報酬扣內扣自行複利。

確定性（直線）模式**未**納入：通膨、稅負、交易手續費、報酬波動與順序風險、各標的報酬差異。真實世界三檔報酬不會相同，且會有大幅上下震盪——直線模式僅供理解「量級」與「相對關係」。

### 蒙地卡羅模式

在「進階設定」切換到「參數化」後改用機率化模型，補上直線模式忽略的**波動度**與**報酬順序風險**：

- 把「年化報酬」視為**算術平均** μ，扣掉配置加權平均內扣得到投組淨報酬。
- 在投組總額層級，每月從常態分布 `Normal(μ/12, σ/√12)` 抽取報酬並複利，跑數百至數千條路徑。
- 輸出各年 **P10／P50／P90** 機率帶、保本機率（最終 ≥ 投入本金的比例）與翻倍機率。
- 顯示**隱含幾何（CAGR）報酬 ≈ μ − σ²/2**：波動越大，實際複利越被「波動拖累」侵蝕，避免只看算術平均而高估。
- 一致性：當波動度 σ=0 時，三條分位數會重合並回到直線試算結果。

蒙地卡羅仍是簡化模型（常態分布、固定 μ/σ、各年獨立、無動態相關性與肥尾），結果為機率區間而非保證。

## 資料更新

- **股價與殖利率**：按標的表上方的「更新股價」會並行抓取上市（TWSE `STOCK_DAY_AVG_ALL`）、上櫃（TPEx）**最近收盤價**，以及上市殖利率／本益比（TWSE `BWIBBU_ALL`）自動回填殖利率。皆免註冊、免金鑰，非盤中即時。若大盤來源全數失敗，會自動改用 **FinMind 逐檔備援**（免 token）；仍失敗則提示可手動填入。狀態列會標示**來源與擷取日期**。
- **抓價備援順序**：每個來源依序嘗試「直連 OpenAPI → corsproxy.io → allorigins」繞過瀏覽器跨域（CORS）；大盤全失敗再走 FinMind 逐檔。若想要最穩定，可自架 **Cloudflare Worker** 白名單代理，把上述 OpenAPI 端點轉發並補上 CORS 標頭，再把前端抓取網址改指向該 Worker。
- **名稱、股價、殖利率、波動%**：新增標的輸入代號後會自動帶入（FinMind，免金鑰）；波動%以約 14 個月日收盤的日對數報酬標準差 ×√252 估算。
- **內扣費用（總費用率）**：無免金鑰公開 API，改由程式內建費率表自動帶入並鎖定欄位——常見台股 ETF 採公開資料估值、個股為 0、未收錄的 ETF 給 0.4% 保守估值；新增/重抓代號或按「更新股價」會一併刷新。費率每年會微調，數字僅供規劃參考；要更精準可於原始碼 `FEE_TABLE` 補上正確值。

## 部署到 GitHub Pages

有兩種方式，擇一即可：

### 方式一：直接從分支部署（最簡單，零建置）

1. 將 `index.html` 放進 repo 根目錄（或 `/docs`）。
2. 進入 repo → **Settings → Pages**。
3. Source 選 `Deploy from a branch`，branch 選 `main`，資料夾選 `/ (root)`（若放在 docs 則選 `/docs`）。
4. 儲存後等待一兩分鐘，即可透過上方網址開啟。此方式直接服務原始 `index.html`（瀏覽器端 Babel）。

### 方式二：GitHub Actions 自動建置部署（載入更快，無 Babel）

repo 已內建 `.github/workflows/deploy.yml`：推送到 `main` 時會自動 `npm run build` 並把預編譯後的 `dist/`（已移除瀏覽器端 Babel）部署到 Pages。

1. 進入 repo → **Settings → Pages**。
2. Source 改選 **`GitHub Actions`**。
3. 之後每次推到 `main`，Actions 會自動建置並上線；也可在 Actions 頁面手動執行（workflow_dispatch）。

> 兩種方式只能擇一（Pages 的 Source 是單選）。想要最快首屏就用方式二，想要零設定就用方式一。

## 其他部署平台（Zeabur／Cloudflare Pages／Netlify／Vercel 等）

本工具為純靜態頁面，上述平台皆可部署，但對單檔 HTML 而言**並非必要**——GitHub Pages 已足夠。只有在你之後需要**後端／serverless**（例如為盤中即時報價架設繞過 CORS 的 proxy）或想要亞洲機房低延遲時，才值得考慮 Zeabur 這類 PaaS。靜態檔案的部署設定（建置指令 `npm run build`、輸出目錄 `dist`）與方式二相同。

## 技術說明

- 單一 `index.html` 即可直接開啟使用，無強制打包流程。
- React 18 + Recharts 透過 CDN（unpkg、已釘確切版本）載入；預設 JSX 由 Babel standalone 於瀏覽器端即時編譯。
- 首次載入需連網並稍候一兩秒；瀏覽器端 Babel 會在 console 出現效能提示，不影響使用。
- 設定持久化使用瀏覽器 `localStorage`，資料僅存於本機、不上傳。
- 載入時對 `localStorage` 舊資料做欄位驗證與補齊；CDN 載入失敗或 render 出錯時會顯示可讀訊息，不會卡在空白「載入中」。

### 選用建置（移除瀏覽器端 Babel，載入更快）

想要最快首屏，可預先把 JSX 編譯成純 JS：

```bash
npm install      # 安裝 esbuild（需連網）
npm run build    # 產出 dist/index.html（不再載入 Babel）
```

`dist/index.html` 仍透過 CDN 載入 React/Recharts，但移除了 ~2.5MB 的 Babel 與瀏覽器端編譯，首屏明顯更快。原始 `index.html` 不受影響、仍可直接使用。`node_modules/` 與 `dist/` 已列入 `.gitignore`。

### 安全性（SRI）

CDN `<script>` 已釘確切版本並加上 `crossorigin`。如需防供應鏈竄改，可另外為各 `<script>` 加上 `integrity`（SRI）雜湊；雜湊需由實際檔案產生，例如：

```bash
curl -s https://unpkg.com/react@18.3.1/umd/react.production.min.js | openssl dgst -sha384 -binary | openssl base64 -A
```

將輸出填入對應 `<script>` 的 `integrity="sha384-..."`。改版時記得一併更新。
